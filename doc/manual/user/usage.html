SqlParser - Users' Manual - Usage

Using the Binaries of the Project

Standard SQL:2008 for JDBC

The typical "users" of the SqlParser binaries are developers who want to implement a façade of the JDBC driver of a proprietary JDBC implementation of a DBMS.

They will usually extend some of the parser classes based on SqlBase and override their creation in an extension of SqlFactory. In the parser classes they will typically override the format() method to produce proprietary SQL. Then they will override the JDBC method Connection.nativeSQL() by first parsing and then formatting the given standardized SQL input. The various execution methods of the JDBC Statement (incl. PreparedStatement) class then are modified to always execute the native SQL version of the given standard SQL:2008 statement.

For information about the parser classes, the Javadoc can be consulted. For more extensive information the sources can be downloaded, in order to study the JUnit tests of all classes.

Evaluation of SQL queries

Another use of the SqlParser binaries is the construction of a JDBC driver for databases lacking a reasonable JDBC implementation. The project JdbcAccess demonstrates this approach. The evaluation of a query is currently limited to non-nested single-table queries. For each row in a table the corresponding column values are set in the TablePrimary class of an SqlStatement. Then the resulting values of the SELECT query are evaluated in the SelectSublist classes of the same query.

Incomplete Parser

SqlParser only addresses the issues that were most important for its use in the SIARD project. It can and should be extended to reach a completely standardized way of accessing any type of relational database. So users of SqlParser are encouraged to become developers and submit enhancements to the project.
